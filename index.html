introduction

The EBS Integration Kit from LikeMinds Consulting is a software solution which enables Single Sign-On & Just-In-Time Provisioning for the Oracle E-Business Suite ERP. EBS Integration Kit (EIK) is developed to work with PingFederate, an Identity and Access Management solution from PingIdentity. The EIK works along with PingFederate for enabling Single Sign-On & provisioning while using a modernized approach to integrate with PingFederate via the OpenID Connect protocol

purpose

The purpose of this document is to summarize the release updates for EBS Integration Kit 2.3.0.2. The release updates summarize the changes with respect to current & previous product features.

scope

The EIK 2.3.0.2 release updates provided are tested & certified against the following versions of Oracle and PingIdentity products.

release update EBS integaration kit 2.3.0.2

The EBS Integration Kit 2.3.0.2 comes with more features with a more modernized approach while supporting a wider range of versions of Oracle E-Business Suite

  1. The EBS Integration Kit 2.3.0.2 has extended its interoperability with Microsoft Azure AD as the OIDC provider. Therefore, Azure AD solution can be extended to on-prem Oracle E-Business Suite solutions.
  2. The EIK 2.3.0.2 integrates with PingOne Enterprise as a token provider. EIK 2.3.0.2 can be deployed with a complete PingIdentity Suite (PingAccess + PingFederate + PingOne Enterprise)
  3. The EIK 2.3.0.2 integrates with PingOne Enterprise as a token provider. EIK 2.3.0.2 can be deployed with a complete PingIdentity Suite (PingAccess + PingFederate + PingOne Enterprise)
  4. EIK 2.3.0.2 now supports PingFederate 11.0.2
  5. EIK can now obtain claims from both the ID token and UserInfo endpoint

enhancement

    update logic 4j2 libaries
  1. EIK 2.3.0.2 uses Log4j Version 2.17.0 to address the vulnerabilities stated in CVE-2021-44228, CVE-2021-45046, and CVE-2021-45105.
    2. Externalized JIT provisioning
  2. New JIT provisioning SDK Properties are externalized.
    3. Externalized CACERTS
  3. The cacerts file has been externalized.
    4. Externalized security. properties
  4. Added functionality to set Security Headers and SameSite value in Cookies generated. Addressed formatting issues and issues in Cookie Service for setting multiple cookies and handling exception cases where state Cookie validation is enabled and configurable.
